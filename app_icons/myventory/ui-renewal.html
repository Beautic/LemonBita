아이콘이 곧
홈 화면이다

픽셀 M 아이콘은 격자 슬롯에 아이템이 담긴 모습입니다. 그런데 지금 홈 화면은 흰 배경에 누끼 이미지가 둥둥 떠 있어요. 둘이 같은 언어를 쓰지 않습니다. 이 문서는 그 간극을 메웁니다.

아이콘 = 슬롯 격자 + 강조된 한 칸.
홈 화면도 정확히 그래야 합니다. 아이템은 슬롯 안에 놓이고, 즐겨찾기는 붉은 칸이 되고, 빈 슬롯은 “추가”가 됩니다. 그러면 앱을 열자마자 아이콘이 살아 움직이는 것처럼 보여요.

03

바꿀 것 — 우선순위 순

위 3개가 임팩트의 80%입니다. 나머지는 정리 작업이에요.

01 그리드를 슬롯으로 세로 카드(0.6) → 정사각 슬롯. 배경 채움 + 모서리 6px + 얇은 테두리. 이 하나가 리뉴얼의 전부라 해도 됩니다. 정사각이라야 피규어·말랑이가 어색하지 않고, 누끼 이미지도 슬롯 배경 위에서 안정적으로 보여요. childAspectRatio: 0.6→1.0 +Container(color: slot, radius: 6) home_screen.dart · _buildClothingGridItem (~710행) 임팩트 최대
02 빈 슬롯 = 아이템 추가 그리드 마지막에 점선 슬롯 + “+”. 아이콘의 빈 칸과 정확히 대응하고, “아직 자리가 남았다”는 수집욕을 자극합니다. 지금은 추가 버튼이 하단 탭에만 있어 발견성이 낮아요. itemCount + 1→마지막 셀에 EmptySlot 신규
03 즐겨찾기 = 붉은 모서리 아이콘의 빨간 칸과 대응하는 기능. 현재 즐겨찾기 개념이 없습니다. “최애 아이템”은 확장 컨셉(피규어·말랑이 = 애착)의 핵심 감정인데 비어 있어요. clothes.isFavorite: bool+슬롯 우상단 삼각 마커 신규
04 원형 스토리 칩 → 사각 칩 원형은 인스타 “사람” 언어. 사각 슬롯 아이덴티티와 충돌합니다. CircleAvatar 56×56→Chip(radius: 4) UI
05 파랑 제거 · 액센트 하나로 “세탁 필요”가 Colors.blueAccent. 팔레트에 없는 색이고, 빨강과 경쟁합니다. 빨강 하나로 통일하고, 세탁 배지는 모서리 점 하나로 축소. blueAccent→#DE3B26 UI
06 수치는 전부 모노스페이스 아이템 개수 · 착용 횟수 · 날짜 · 등록일. 인벤토리 톤을 만드는 건 색이 아니라 이 서체입니다. fontFeatures: tabular-nums UI
07 lib/theme/ 신설 지금 테마는 main.dart 20줄이 전부고 색은 13,000줄에 흩어져 있습니다. 위 항목들을 하나씩 고치는 것보다 이걸 먼저 하는 게 결국 빠릅니다. 순수 흑백 → 따뜻한 중성색으로 바꾸는 것도 여기서 한 번에. #FFFFFF / #000000→#F2F1ED / #121213 기반
08 radius 스케일 고정 현재 8 / 12 / 16 / 20 / 24가 뒤섞여 있습니다. 슬롯 6 · 카드 10 · 버튼 12 · 시트 20으로 고정. 정리
09 카피 중립화 MY CLOSET → MYVENTORY · "옷 정보 없음" → "정보 없음" · "나만의 디지털 옷장" → "내 아이템 인벤토리" · "옷장에 추가하기" → "인벤토리에 추가" 카피
바꾸지 말 것. 하단 4탭 구조 · 흑백 기조 · 누끼(배경제거) · 폴더 시스템 — 전부 잘 작동하고 있습니다. 리뉴얼은 구조와 언어를 바꾸는 것이지, 갈아엎는 게 아니에요. 번들 ID com.antigravity.dress도 그대로 (바꾸면 기존 사용자 유실).
기능은 아이템 타입을 따라간다. OOTD · 코디 캔버스 · 세탁 주기 · 날씨 추천은 옷 전용 기능입니다. 지우지 말고 의류 카테고리에서만 노출되게 하세요. 피규어 폴더를 보고 있는데 “오늘 뭐 입지”가 뜰 이유는 없으니까요. 이게 확장 컨셉의 유일한 규칙입니다.
