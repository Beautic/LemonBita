# 디지털 옷장 (My Digital Closet) 작업지시서 v4.3

본 문서는 **v4.2 작업지시서** 위에서 추가로 설계 및 구현된 **피드백 카운팅 표시, 코디 아이디어 폴더링(그룹핑) 및 관리 기능, 한글 자소분리(NFD) 에셋 매칭 수정, 모바일 뒤로가기 강제 종료 방지** 내역을 세세하게 기록합니다.

사용자 피드백에 맞춰 조작의 간편성과 에셋 노출의 안정성, 그리고 모바일 사용성을 대폭 강화했습니다.

---

## 1. 데이터 모델 및 스키마 변경

소셜 인터랙션의 인지도를 개선하고 코디 아이디어의 분류 편의성을 제공하기 위해 Firestore 스키마와 보안 규칙을 설계 및 적용했습니다.

### 1.1 Planned Folders (코디 폴더) 컬렉션 추가
사용자가 코디 아이디어를 주제(예: "유럽여행", "데일리")별로 분류할 수 있도록 폴더 컬렉션을 추가했습니다.
```javascript
planned_folders/{folderId} {
  userId: string,       // 생성한 사용자의 UID
  name: string,         // 폴더 이름 (예: '유럽여행')
  createdAt: timestamp  // 폴더 생성 일시
}
```

### 1.2 Planned OOTDs (코디 아이디어) 스키마 확장
코디 아이디어가 속할 폴더를 연결할 수 있도록 `folderId` 필드를 추가했습니다.
```javascript
planned_ootds/{docId} {
  // ... 기존 필드
  folderId: string      // (옵션) planned_folders 컬렉션의 문서 ID. 지정되지 않거나 미분류일 경우 null 혹은 필드 삭제.
}
```

### 1.3 Firestore 보안 규칙 (firestore.rules) 업데이트
새로운 컬렉션 추가에 따라 본인이 생성한 폴더 데이터에 대해서만 접근 권한을 획득하도록 규정했습니다.
```javascript
// 예비 OOTD 폴더 (Planned Folders)
match /planned_folders/{folderId} {
  allow read: if request.auth != null && (resource == null || resource.data.userId == request.auth.uid);
  allow create: if request.auth != null && request.resource.data.userId == request.auth.uid;
  allow update, delete: if request.auth != null && resource.data.userId == request.auth.uid;
}
```

## 2. 핵심 화면 및 컴포넌트 구현 상세

### 2.1 게시글 피드백(좋아요/댓글) 수 아이콘 옆 직접 카운팅 표시
* **개편 사항**: 기존 화면 최하단에 텍스트 형태로 노출되던 "좋아요 X개 · 댓글 Y개" 정보 영역을 전면 제거했습니다.
* **디자인 고도화**: 하트(좋아요) 및 댓글 아이콘 우측에 직접 숫자가 나오도록 병렬 Row 배치했으며, 개수가 0개일 때는 생략하고 1개 이상일 때만 숫자가 나타나도록 최적화했습니다. 탭 영역을 충분히 확보하는 `InkWell` 제스처를 적용했습니다.

### 2.2 코디 아이디어 그룹핑 및 폴더 관리 시스템
* **폴더 바 구현**: '코디 아이디어' 탭 최상단에 "전체", "미분류", 사용자가 추가한 "폴더 칩(Chip) 목록" 및 "새 폴더 만들기" 아이콘 버튼이 위치한 가로 스크롤 형태의 폴더 바를 구현했습니다. 
* **쿼리 최적화 (복합 색인 에러 해결)**: Firestore에 복합 색인이 배포되어 있지 않아 쿼리가 오류를 반환하던 문제를 해결하기 위해, 데이터베이스 조회 시 `orderBy` 필터를 제거하고 **Dart(로컬) 메모리 단에서 데이터를 정렬**하도록 최적화했습니다. 이를 통해 로딩 중단 없이 실시간 폴더 목록을 조회할 수 있습니다.
* **폴더 이름 수정 및 삭제 모달**: 폴더 칩을 길게 누르면(Long Press) 하단 모달 시트("폴더 관리")가 떠서 폴더의 이름을 즉시 수정하거나 폴더를 안전하게 삭제(해당 폴더 내 코디는 '미분류'로 복구)할 수 있는 관리 흐름을 완벽히 구축했습니다.
* **아이디어 상세화면 & 목록 화면 폴더 분류 연동**:
  * **상세화면**: 우측 상단 앱바에 폴더 이동 아이콘을 도입해, 라디오 버튼 형태의 다이얼로그로 원하는 폴더를 손쉽게 변경할 수 있습니다.
  * **목록화면**: 목록 뷰의 각 코디 카드를 길게 누르면(Long Press) 상세화면으로 들어갈 필요 없이 즉시 폴더 이동 팝업 다이얼로그가 떠서 신속한 일괄 분류 및 정리가 가능하게 설계했습니다.

## 3. UI/UX 및 모바일 최적화 상세

### 3.1 NFD/NFC 인코딩 불일치를 해결하는 유니코드 Hex 에셋 시스템
* **배경**: Mac OS의 파일 시스템(자소 분리 NFD) 특성과 브라우저의 리소스 요청 방식(자소 합침 NFC)이 충돌하여, '원피스' 카테고리를 포함한 일부 한글 파일명의 에셋이 웹 빌드 후 404 에러와 함께 폴백 아이콘으로 깨져서 출력되던 현상을 발견했습니다.
* **해결 방안**: 한글 파일명 자체를 완전히 배제하도록 설계를 전환했습니다.
  1. `assets/icons/` 내의 모든 파일명을 유니코드 코드 포인트 16진수 문자열의 영문화 파일명(예: `가디건.png` ➔ `ac00_b514_ac74.png`, `청바지.png` ➔ `ccad_bc14_c9c0.png`)으로 강제 일괄 변환하여 정규화했습니다.
  2. `CategoryData.getIconPath` 함수에서 한글 소분류 명의 공백(' ') 및 슬래시('/') 기호를 선제적으로 제거한 후, 각 글자의 유니코드 값을 조합해 영문화 hex 파일명을 정확하게 매칭하도록 수정했습니다.
  3. Git Config 설정(`core.precomposeunicode true`)을 강제 적용해 한글 자소분리로 인한 추적 유실을 완벽 방지했습니다.

### 3.2 모바일 뒤로가기 제어를 통한 강제 앱 종료 방지 (PopScope)
* **목적**: 모바일 웹이나 스마트폰 환경에서 사용자가 습관적으로 뒤로가기 버튼/제스처를 누르면 앱이 예고 없이 즉각 꺼져버리는 불편한 사용성 개선.
* **구현 로직**: 
  * `MainScreen` 최외각에 Flutter 최신 표준인 `PopScope` 위젯을 적용해 물리적 뒤로가기 이벤트를 탈취했습니다.
  * 사용자가 'OOTD'나 '프로필' 등 다른 탭에 있을 때는 홈 탭인 **'옷장' 탭으로 먼저 강제 복귀**하도록 유도했습니다.
  * '옷장' 탭에 위치한 상태에서 뒤로가기를 입력하면 **"뒤로가기를 한 번 더 누르면 종료됩니다."**라는 스낵바 메시지를 노출합니다.
  * 안내가 나간 후 **2초 이내에 다시 뒤로가기를 입력할 경우에만** `SystemNavigator.pop`을 호출하여 비로소 앱을 종료시킴으로써 사용자 실수를 미연에 방지했습니다.

## 4. 빌드 및 배포 명령어
* **전체 배포 명령어**:
```bash
git add -A && export PATH="$HOME/fvm/versions/3.16.9/bin:$PATH" && flutter build web --no-tree-shake-icons && ./firebase_bin deploy --only hosting
```
* **Firestore 규칙 배포**:
```bash
./firebase_bin deploy --only firestore:rules
```
