# 실험: 구글 이미지 검색 연동 작업지시서

이 문서는 기존 '나만의 디지털 옷장' 애플리케이션에 **구글 이미지 검색 기능**을 통합하기 위한 실험적 컴포넌트 개발 작업지시서입니다. 코드를 직접 생성하기 전, 구현 방안과 요구사항을 정의합니다.

## 1. 실험 개요
- **목적**: 사용자가 자신의 옷장(앱)에 등록된 옷 사진이나 태그 정보를 바탕으로 구글 이미지 검색을 자동으로 실행하여 비슷한 스타일, 코디, 또는 구매처를 쉽게 찾을 수 있도록 합니다.
- **방식**: 앱 내에서 브라우저를 직접 띄우거나(`WebView`), 기기의 기본 브라우저를 호출(`url_launcher`)하여 미리 정의된 구글 이미지 검색 파라미터가 포함된 URL로 연결합니다.

## 2. 연동 방식 분석 및 URL 구조
구글 이미지 검색 사이트(`https://images.google.com/?hl=ko`)를 활용하여 자동 검색을 수행하기 위해 다음 두 가지 방식을 지원합니다.

### A. 이미지 기반 역검색 (사진 활용)
- 현재 구글의 모바일/웹 이미지 역검색은 **Google Lens**로 통합되어 서비스되고 있습니다.
- Firestore에 이미 업로드되어 있는 공개된 `imageUrl`을 쿼리로 활용합니다.
- **URL 패턴**: `https://lens.google.com/uploadbyurl?url={업로드된_Firebase_Storage_이미지_URL}&hl=ko`
- **장점**: 사용자가 찍은 옷 사진과 형태, 색상, 패턴이 유사한 상품을 가장 정확하게 찾아냅니다.

### B. 텍스트 기반 이미지 검색 (태그, 브랜드 활용)
- 사용자가 기입한 해시태그(`tags`)나 브랜드명(`brand`)을 조합하여 검색어로 사용합니다.
- **URL 패턴**: `https://www.google.com/search?tbm=isch&q={검색어}&hl=ko`
- **예시**: `https://www.google.com/search?tbm=isch&q=나이키+반팔티+여름옷&hl=ko`

## 3. 필요 기술 스택 및 패키지
- **패키지**: `url_launcher` (버전: 최신 안정화 버전)
  - 앱에서 외부 웹 브라우저 앱(Safari, Chrome 등)을 안전하고 원활하게 열기 위해 사용합니다. 웹 환경(Flutter Web)에서도 새 탭으로 여는 기능을 완벽히 지원합니다.

## 4. UI/UX 요구사항
기존 앱의 코드베이스를 유지하면서, 옷 상세 정보 화면(`ClothingDetailScreen`)에 새로운 버튼을 추가하여 기능을 노출합니다.

- **위치**: `ClothingDetailScreen` 스크롤 하단, 기존 '정보 저장하기' 버튼과 나란히 배치하거나 그 아래에 배치합니다.
- **디자인 (Component UI)**:
  - 기존의 주 버튼(`FilledButton`)과 시각적인 차이를 두기 위해 테두리만 있는 `OutlinedButton` 또는 톤 다운된 `FilledButton.tonal`을 사용합니다.
  - **아이콘**: `Icons.search_rounded` 또는 `Icons.travel_explore`
  - **라벨**: "구글에서 비슷한 옷 찾기"
- **동작 흐름**:
  1. 버튼 탭 시, 하단에서 `BottomSheet` 또는 화면 중앙에 `AlertDialog`가 나타나 두 가지 옵션을 제시합니다.
     - 1옵션: **"이 사진으로 검색하기"** (이미지 역검색 URL 호출)
     - 2옵션: **"태그/브랜드로 검색하기"** (텍스트 기반 이미지 검색 URL 호출)
  2. 사용자가 옵션을 선택하면, 생성된 URL을 `url_launcher`의 `launchUrl` 함수에 전달합니다.
  3. 브라우저 앱이 실행되며 구글 검색 결과 창이 띄워집니다.

## 6. 구현 시 주의사항 (바이브코딩 지침)
- **URL 인코딩**: 텍스트로 검색할 때나 이미지 URL을 파라미터로 넘길 때, `Uri.encodeComponent()` 등을 사용하여 URL 파싱 오류(띄어쓰기, 특수문자 등)가 발생하지 않도록 주의해야 합니다.
- **예외 처리**: 만약 `url_launcher`가 기기 환경 문제로 브라우저를 열지 못하는 경우(`canLaunchUrl`이 false를 반환하는 경우), "브라우저를 열 수 없습니다."라는 `SnackBar`를 띄워야 합니다.
- **플랫폼 설정**: `url_launcher`를 정상 작동시키기 위해 Android의 `AndroidManifest.xml`과 iOS의 `Info.plist` 파일에 `https` 스킴(scheme) 쿼리를 반드시 허용해야 합니다.
