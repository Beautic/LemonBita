# 디지털 옷장 (My Digital Closet) 작업지시서 v4.4

본 문서는 **v4.3 작업지시서** 상에 기술된 설계 및 구현 사항을 바탕으로, 최종 검증과 한글 에셋 파일시스템 인코딩(NFD) 문제의 완전한 해결, 그리고 모바일 물리 뒤로가기 제스처 대응(PopScope)을 정식 배포 버전 수준으로 끌어올린 **v4.4 정식 사양**을 정의하고 변경 사항을 기록합니다.

---

## 1. 개요 및 배경

기존 디지털 옷장 서비스의 실사용성과 모바일/웹 크로스 플랫폼 호환성을 보완하기 위해 다음과 같은 한계점을 해결했습니다.
1. **소셜 인터랙션 인지도 부족**: OOTD 피드 및 상세 화면에서 하트/댓글 개수를 보기 위해 화면 하단까지 스크롤해야 하던 불편함을 개선했습니다.
2. **코디 관리 편의성 부재**: 추천받은 가상 코디 아이디어가 누적됨에 따라, 이를 폴더별(예: "유럽여행", "데이트룩")로 그룹핑하고 실시간으로 관리(수정/삭제)할 필요성이 대두되었습니다.
3. **Firestore 복합 색인 미생성 오류**: 서버 측 인덱스 설정 없이도 즉각적인 실시간 데이터 정렬 및 폴더 필터링을 지원해야 했습니다.
4. **한글 자소분리(NFD)에 따른 에셋 로드 실패**: Mac OS 파일시스템(NFD)과 브라우저 인코딩(NFC) 불일치로 인해 특정 카테고리(원피스 등)의 한글 파일명 아이콘 이미지가 웹에서 404 에러와 함께 깨지는 문제를 해결했습니다.
5. **모바일 뒤로가기 제스처 시 앱 강제 종료**: 스마트폰 브라우저나 앱 환경에서 뒤로가기를 입력할 경우 앱이 전면 종료되는 대신 홈 화면 복귀 및 이중 확인 절차를 거치도록 변경했습니다.

## 2. 세부 설계 및 구현 내용

### 2.1 OOTD 및 코디 피드백 카운팅 UI 개선
* **위치**: `lib/widgets/ootd_interaction_bar.dart`
* **구현 방식**:
  - 기존 상세 화면 하단의 텍스트 방식 개수 노출 영역을 제거했습니다.
  - 좋아요(하트) 및 댓글 아이콘 바로 우측에 카운팅 숫자를 배치하는 Row 레이아웃을 적용했습니다.
  - 피드백 개수가 0개일 때는 레이아웃의 직관성을 위해 숫자를 숨기고, **1개 이상일 때만** 개수를 숫자로 표시하도록 예외 처리를 추가했습니다.
  - 사용자가 실수 없이 터치할 수 있도록 충분한 패딩 영역을 확보한 `InkWell` 제스처 구조를 사용합니다.

### 2.2 코디 아이디어 폴더링 및 실시간 정렬
* **데이터 모델**: 
  - `planned_folders` 컬렉션: `userId`, `name`, `createdAt` 필드로 구성.
  - `planned_ootds` 컬렉션: `folderId` 필드를 추가하여 특정 폴더에 바인딩.
* **보안 규칙 (`firestore.rules`)**:
  - 본인이 소유한 폴더에 대해서만 CRUD 권한이 할당되도록 강제하는 규칙을 추가했습니다.
* **UI 컴포넌트 (`lib/screens/ootd_screen.dart`)**:
  - 최상단에 "전체", "미분류", "사용자 정의 폴더 목록"이 나타나는 가로 스크롤형 폴더 칩 바를 구성했습니다.
  - **Firestore 복합 색인 우회**: 서버에 복합 색인을 배포하는 번거로운 과정 없이, `planned_ootds` 문서를 전체 조회한 후 Dart 로컬 메모리 단에서 `createdAt` 기준으로 `.sort()`를 수행하여 쿼리 실패를 원천적으로 방지했습니다.
  - **폴더 이동 다이얼로그**: 코디 카드나 상세 화면(`lib/screens/planned_ootd_detail_screen.dart`)에서 폴더 아이콘을 누르거나 꾹 누르는(Long Press) 제스처 발생 시, 라디오 버튼 목록이 포함된 다이얼로그가 팝업되어 빠르게 폴더를 지정할 수 있게 했습니다.
  - **폴더 이름 수정 및 삭제**: 폴더 칩을 꾹 누르면 하단 시트가 발생하여, 즉시 폴더 이름을 변경하거나 폴더를 삭제할 수 있습니다. 폴더 삭제 시 해당 폴더의 코디들은 안전하게 '미분류(folderId = null)' 상태로 격하되어 데이터 유실을 방지합니다.

### 2.3 한글 에셋 깨짐 방지를 위한 16진수 유니코드 인코딩 시스템
* **에셋 리네이밍 스크립트 (`scripts/rename_to_hex.py` & `scripts/normalize_assets.py`)**:
  - `assets/icons/` 내의 한글 파일명을 유니코드 16진수 영문화 코드 포인트 형태(예: `원피스.png` ➔ `c6d0_d53c_c2a4.png`)로 강제 변경했습니다.
  - Mac OS 파일시스템이 NFC와 NFD 파일명을 동일하게 매칭하는 점을 감안하여 중복 파일 처리에 따른 원본 덮어쓰기/유실 문제를 완벽히 방지하는 안전화 리네임 알고리즘을 사용했습니다.
* **카테고리 매칭 로직 (`lib/utils/categories.dart`)**:
  - `CategoryData.getIconPath` 메서드에서 소분류 명칭을 한글 유니코드 16진수 문자열로 변환하여 에셋 파일 경로를 탐색하게 함으로써, 어떤 웹 브라우저 환경에서도 404 없이 에셋 이미지가 100% 정상 출력되도록 설계했습니다.

### 2.4 모바일 뒤로가기 제어 (`PopScope`)
* **위치**: `lib/screens/main_screen.dart`
* **동작 순서**:
  1. 사용자가 다른 탭(OOTD, 친구, 프로필 등)에 위치한 상태에서 물리 뒤로가기를 입력할 경우, 홈 탭(옷장 탭)으로 네비게이션을 즉각 고정시킵니다.
  2. 홈 탭에 위치한 상태에서 1차 뒤로가기 입력 시, 화면 하단에 **"뒤로가기를 한 번 더 누르면 종료됩니다."**라는 스낵바 메시지를 표시합니다.
  3. 스낵바 안내가 활성화된 시점부터 **2초 이내에 다시 한번 뒤로가기를 입력할 경우에만** `SystemNavigator.pop()`을 트리거하여 최종적으로 앱을 안전하게 종료시킵니다.

## 3. 배포 및 검증 결과

* **배포 환경**: Firebase App Hosting & Static Hosting
* **배포 명령어**:
  ```bash
  git add -A && flutter build web --no-tree-shake-icons && ./firebase_bin deploy --only hosting
  ```
* **검증 내용**:
  - 모바일 제스처를 통한 안전 종료 로직이 스마트폰 웹에서 오작동 없이 수행됨을 확인했습니다.
  - '원피스' 카테고리를 포함한 모든 옷 카테고리 아이콘이 404 리소스 에러 없이 투명하고 선명하게 렌더링됩니다.
  - 코디 상세 피드백 아이콘 옆에 좋아요와 댓글 개수가 동적으로 렌더링되며, 실시간 업데이트가 이루어집니다.
